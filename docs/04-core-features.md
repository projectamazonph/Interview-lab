# Amazon VA Interview Lab - Core App Features

## 4.1 Interview Question Bank

The question bank should be searchable and filterable by:

| Filter | Options |
|--------|---------|
| Role | PPC VA, Account VA, Listing VA, Reporting VA, Agency VA |
| Difficulty | Beginner, intermediate, advanced |
| Question type | Behavioral, technical, scenario, case study, tool-specific |
| Skill | PPC, Excel, Seller Central, client communication, reporting |
| Answer format | STAR, bullet answer, case response, calculation |
| Time limit | 30 seconds, 1 minute, 3 minutes, 5 minutes |

Each question should include:
| Field | Description |
|-------|-------------|
| Question | The actual interview question |
| Why employers ask this | Hiring intent |
| Strong answer should include | Key points |
| Weak answer signs | Red flags |
| Sample answer | Optional |
| Practice mode | Voice, typed, flashcard, mock interview |
| Rubric | Scoring criteria |

## 4.2 AI Mock Interview Simulator

### Modes:
| Mode | Description |
|------|-------------|
| Quick drill | 5 random questions |
| Role interview | 20-minute simulated interview |
| Technical screen | PPC/reporting-heavy |
| Client communication interview | Scenario-based |
| Final interview | Mixed behavioral and technical |
| Practical test debrief | User explains how they solved a case |

The AI should ask follow-up questions like a real interviewer:
> "You mentioned ACoS. How would you explain the difference between ACoS and ROAS to a client?"
> "You said you would add negative keywords. What threshold would you use before making that decision?"
> "A campaign has 20 clicks and 0 orders. Would you pause it immediately? Why or why not?"

The app can teach threshold logic: promoting terms after sufficient clicks and conversions, negating poor performers, and using stricter thresholds for competitive categories.

## 4.3 Answer Coach

The answer coach rewrites weak answers into stronger, truthful answers.

**Example:**
User answer:
> "I know Amazon PPC and can optimize campaigns."

Improved answer:
> "I understand the basic PPC workflow: research campaigns discover search terms, performance campaigns focus on proven keywords, and optimization involves reviewing CTR, CPC, CVR, ACoS, and ROAS. In a junior role, I would follow the team's SOPs, export search term reports, flag high-click zero-sale terms, and document all bid or negative keyword recommendations before implementation."

The coach should never invent experience. It can help users say:
> "I have trained on this workflow."
or
> "I understand the process and can execute it under supervision."

instead of falsely saying:
> "I managed $50,000/month in ad spend."

## 4.6 Practical Assessment Simulator

### Practical tests:
| Assessment | What user does |
|------------|----------------|
| Search Term Report Review | Identify promote, pause, negate, monitor keywords |
| Campaign Naming Exercise | Build compliant campaign names |
| ACoS/ROAS Calculation | Calculate metrics from spend/sales data |
| CPC Ceiling Exercise | Calculate maximum CPC from price, target ACoS, and CVR |
| Listing Readiness Audit | Score images, title, bullets, A+ content, reviews, inventory |
| PPC Client Update | Write a concise weekly performance summary |
| Campaign Launch Plan | Choose which campaign types to launch first |
| Keyword Intent Classification | Classify terms as high/mid/low intent |
| Budget Pacing Scenario | Decide whether to increase, decrease, or throttle budget |
| Interview Case Presentation | Explain decisions out loud |

The campaign launch plan should train candidates on phased rollout: research and branded defense first, then competitor targeting, then SKAG/category/cross-sell expansion, then remarketing/audiences, then seasonal campaigns.
